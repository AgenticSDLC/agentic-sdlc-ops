# Non-Goals

Phase 1 of `agentic-sdlc-ops` does not attempt to:

- centralize execution infrastructure across repositories
- require a single CI provider or deployment platform
- require a single package manager or language ecosystem
- mandate browser smoke tests for repositories that do not have browser flows
- replace repository-specific architecture documentation
- encode environment-specific shell or authentication wrappers into the core standard
- broaden issue execution into a full project management framework

This repository is a reusable operating kit, not a monolithic platform.
